# 产品需求文档

## 问题

现有页面只有 14 条人工 JSON，采集脚本只有少量脆弱的正则抓取。它无法判断“行业热点”，无法区分事实与观点，无法合并同一事件，也没有管理、审核、解释评分或稳定发布能力。

## 产品定义

Agent Pulse 是面向 AI 从业者、产品负责人、创业者和投资观察者的行业情报时间轴。它回答的不是“今天有哪些链接”，而是：

- 发生了什么，证据是什么？
- 为什么它正在形成行业热点？
- 对技术路线、产业结构和业务机会意味着什么？
- 哪些判断是事实，哪些仍是推断？
- 接下来应该观察什么可验证信号？

## 目标

- 只公开经过收敛的事件，不直接倾倒原始条目。
- 让“可信”和“热门”成为可解释的独立维度。
- 同时覆盖全球与中国语境，避免只观察单一社交平台。
- 重要事件可以在 5 分钟内读完并形成可复述认知。
- 管理员能控制来源、采集、评分、发布、置顶和导出。
- 默认零配置 SQLite，本地或服务端可切换 MySQL。
- 静态公开页可直接托管在 GitHub Pages。

## 非目标

- 不做全文新闻镜像，不存储或公开第三方完整文章。
- 不把纯浏览量、单个大 V 转发或单平台榜单直接等同于热点。
- 不承诺绕过登录、付费墙、验证码或平台反爬。
- 第一阶段不做多租户、付费订阅和复杂权限系统。
- 第一阶段不依赖特定 LLM；没有模型密钥时仍能采集、打分、审核和发布。

## 核心对象

### Signal（信号）

一篇公告、论文、帖子、新闻、release 或聚合器条目。只用于内部证据与聚类。

### Event（事件）

多个相关信号收敛后的公开最小单元。Timeline 只展示 Event。

### Source（信源）

带类型、地区、层级、可信度、时效性、获取方式和启停状态的来源定义。

### Track（主线/支线）

跨事件的叙事轴。一个事件可以同时属于一条主线和多条支线，并在不同视角下拥有不同解释。

默认主线：

- 技术演进：Scaling、推理、Agent、世界模型、具身智能、AI Infra；
- AGI 进展：能力边界、自治程度、可靠性、安全与治理；
- 投资视角：融资、并购、CapEx、算力资产、估值与竞争格局；
- 商业化：收入、成本、定价、分发、留存、商业模式；
- 客户类型：To C、To B、To D（Developer）、To G（Government）；
- 中国追赶：跟随、并跑、领先、受限、出海五种阶段。

支线可以按技术分类、时间、地域、公司、人物、政策和应用场景展开，并与主线共享同一事件证据。

### Actor（角色）

公司、实验室、研究机构、投资机构、人物或监管机构。Actor 保存地区、规模、赛道、阶段和可上牌桌程度，支持观察国内大小厂的发展轨迹。

### Resource（模型获取资源）

官方订阅、官方 API、云平台、合规代理或外部比价入口。公开价格必须来自厂商/平台官方来源；第三方比价只提供入口、风险等级和最后核验时间。

## 热点定义

“热点”必须同时满足：

1. 有至少一个高可信事实信号，或两个相互独立的中可信信号；
2. 在短时间窗口内出现多个独立作者/媒体/社区响应；
3. 至少覆盖两个传播域，理想情况下包含中英文两个语境；
4. 扩散速度、独立参与者数量或持续时间达到阈值；
5. 不是同一媒体矩阵的机械转载。

默认热点阈值：`heatScore >= 70` 且 `confidenceScore >= 60`。只有海外或只有国内证据时可以进入“高关注”，但不标记“跨圈热点”。管理员可以覆盖。

## 信息收敛模板

每个公开事件包含：

- 一句话事实：只写已确认内容；
- 核心摘要：2-4 句，交代对象、动作、关键数字和状态；
- 技术洞察：能力、成本、架构或路线变化；
- 行业判断：对竞争结构、生态、资本或监管的含义；
- 业务价值：谁应该做什么，或暂时不做什么；
- 未来观察：1-3 个可验证信号；
- 关键字：3-8 个；
- 证据：按事实源优先展示，标明来源角色；
- 评分解释：为什么热、为什么可信、为什么重要。

## 公开页交互

- 默认显示最近热点与时间轴，不展示原始 feed。
- 点击 item 在右侧打开详情抽屉，URL hash 可直接分享。
- 支持搜索、主题、影响等级、事件类型和时间窗口过滤。
- 重要信息用影响等级、热点标识、关键数字和关键词高亮。
- 支持 `midnight`、`paper`、`signal` 三套主题，记住用户选择。
- 移动端改为底部/全屏详情，不牺牲可读性。
- 首页默认以 CEO / 大厂投资负责人视角呈现：先看战略变化、资本与商业化，再下钻到技术证据。
- 提供“主线模式”和“时空地图模式”：主线模式按叙事轴播放关键节点；时空模式按国内/海外、公司阶段、技术分类交叉查看。
- 提供“中国追赶雷达”，展示国内角色在同一能力/产品/价格节点上相对海外基准的跟随、并跑或领先状态。
- 提供“模型获取”区，按 To C 订阅、To D API、企业采购和政府采购展示官方入口、价格口径、地区和风险说明，并链接 PriceAI 做进一步购买前比价。

## 管理台

- 仪表盘：来源健康、最近采集、待审、已发布、失败任务。
- 来源：增删改查、启停、权重、层级、获取方式、抓取频率。
- 信号：查阅原始元数据、聚类结果、去重状态和证据角色。
- 事件：编辑收敛内容、评分、状态、置顶、隐藏、发布时间。
- 任务：手动执行采集、聚类、评分、导出；查看执行日志。
- 设置：热点阈值、静态站点标题和默认主题。
- 叙事编排：创建主线/支线，拖动事件顺序，设置里程碑、转折点和对比关系。
- 角色雷达：维护国内外角色、技术位置、商业阶段和“上牌桌”评分，查看角色时间轴。
- 视图工作台：可视化组合筛选器、区块、主题、密度、强调色、默认排序和首页栏目；支持预览后发布。
- 资源中心：维护官方订阅/API/云平台入口、单位价格、地区、风险、核验时间和外部比价链接。

## 验收标准

- 一条相同 URL 不会重复入库；相似标题会进入同一事件候选。
- 一个公开事件至少保留一个可点击原始证据链接。
- 热点评分可以追溯到独立来源数、平台数、地区数和时间衰减。
- 管理员可以在 UI 完成来源启停和事件发布/隐藏。
- 管理员可以在 UI 创建主线、把一个事件编排进多条线，并发布一套首页视图。
- 国内角色目录覆盖已定义的重点厂商，角色详情能显示跟随/并跑/领先的证据节点。
- 模型获取资源区区分官方数据与第三方参考，不把卡网或中转渠道包装成官方入口。
- `DATABASE_URL` 未设置时使用 SQLite；设置 `mysql://` 时使用 MySQL 方言。
- `npm run export` 生成完全静态的 `dist/`，不包含 token、数据库或本机路径。
- 桌面端 item 点击后右侧展开详情；移动端可完整阅读。
- 所有核心模块有单测，API 有集成测试，关键页面通过浏览器烟测。
