# PRD：从自动准备升级为无人值守认知发布

## 1. 当前根因

1. 早期 Timeline 来自少量高价值里程碑 seed，2024-07 至 2026-03 通常每月只有 1–4 个事件；2026-07 因产品重构和论文补录集中达到 19 个，采样口径不一致造成“越靠近今天越密集”。
2. 2026-04 至 2026-06 已存在大量一手 Signal 和 review Event，但自动链路只返回发布候选，不真正发布；大量 Event 仍带占位洞察，无法通过 readiness。
3. 论文日报只消费已公开 research Event，且同一 UTC 日期至少 4 篇才生成。当前可见批次集中在 2026-07-11；7 月 11 日后跨周末，arXiv 官方发布节奏为每周五天，不能伪造 7 月 12 日论文。
4. Scout 生成器只写 inbox，后台仍要求“细看、接受、忽略、发布”，与无人值守运营目标冲突。
5. Data Refresh 每 6 小时运行，但评测低于 60 分不会触发额外更新；GitHub Release 也未与正式 Changelog 版本自动同步。
6. 来源目录有少量海外博客和泛社交入口，却没有显式的国内外核心 AI KOL 身份矩阵，也没有区分可自动采集的个人 RSS 与受限的 X、LinkedIn、微博、即刻账户。

## 2. 产品目标

- Scout 达到质量门禁后直接公开，不再进入人工状态队列；未达到门禁的条目自动归档并保留原因。
- Event 通过既有事实、证据和内容 readiness 后自动发布；阻塞对象留在隔离状态，不等待后台人工确认。
- 后台 Event、Scout 与合并队列改为只读审计视图，不显示接受、忽略、编辑发布或确认合并动作。
- 数据刷新提升为每日 6 次；评测低于 60 分且最近没有刷新时，由独立 Quality Guard 触发一次增量更新。
- 正式版本同时存在于 `package.json`、仓库 Changelog、网站 Changelog；CI 成功后自动创建缺失的 GitHub Release。
- 最近 7 天事件在首页、主线、Timeline 和相关事件中具有一致高亮。
- “中国位置”改为“本土进展”，“中国与全球位置”改为“中国进展与全球坐标”。
- 最近三个完整自然月每月至少有 6 个经过一手证据与完整判断整理的公开 Event；当前月继续由自动采集补充。
- 论文区域明确显示最近三天的批次状态：有内容展示数量，无内容说明周末节奏或等待下一批，不制造空论文。
- 建立 KOL 身份矩阵，覆盖海外与中国的研究、Agent、AI Coding、Infra、产品和投资视角；个人 RSS/Atom 可自动采集，社交平台受限时明确展示 restricted，不绕过登录或反爬。

## 3. 自动化边界

无人值守不等于无门禁：

- Scout 必须绑定已公开 Event，且总分、证据分、置信度和新颖度均达到阈值；否则自动归档。
- Event 只有 `evaluateEventReadiness()` 返回 ready 才能自动发布；占位内容、缺少一手证据、冲突事实和低置信度继续阻塞。
- 来源只有许可明确、非 restricted、最近检查健康、20 次健康检查与 7 天观察全部满足时才自动晋级。
- 事实冲突不自动合并；系统保留两个独立事件并记录冲突，而不是等待人工按钮。
- X、LinkedIn、微博与即刻不绕过登录、WAF 或平台限制；优先采集作者公开 RSS、个人站点或 GitHub Atom。

## 4. 验收目标

- 后台不再出现 Scout 的“细看、接受、忽略、发布”和 Event 的“编辑 / 发布”入口；自动模式不可关闭。
- 新 Scout 可直接成为 `published`，旧 inbox 会被自动发布或归档。
- ready Event 自动写入 `published_at`，blocked Event 不被误发。
- Quality Guard 能解析真实评测；低于 60 分时按冷却规则 dispatch Data Refresh。
- Data Refresh 每 4 小时运行，执行采集、聚类、来源协调、观察/激活、Scout、自动发布、评测、快照和 Pages。
- 正式版本发布后自动生成 GitHub Release，正文来自对应 Changelog 章节。
- 最近 7 天事件有 `data-recent="true"` 和可见的“近 7 天”标识。
- 最近三个完整月密度达到每月 6 个以上；论文批次状态解释 7 月 12 日周末空档与 7 月 13 日等待状态。
- KOL 页面区分自动 feed 与 restricted 社交入口，并至少提供 10 位核心个人的可验证主页或账户。
